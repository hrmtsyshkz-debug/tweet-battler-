# つぶやきバトラー - 開発引き継ぎメモ

Cowork上でのプロトタイピングをClaude Codeなど実開発環境に引き継ぐためのブリーフです。Claude Codeは今回の会話を記憶していないため、このファイルと `tsubuyaki-battler.html` の2つを新しいセッションに渡してください。

## 現状

`tsubuyaki-battler.html` 1ファイルで完結する静的プロトタイプ（ビルド不要、ブラウザで開くだけで動作）。フレームワーク・バックエンドなし。外部依存はCDN経由の `qrcodejs` と `jsQR` の2つのみ。

## 実装済み機能

- SNS投稿のコピペ or 属性選択 → テキストのハッシュ値から決定論的にステータス（HP/攻撃/防御/素早さ）・属性・固有技4つを生成
- 属性相性（8属性、円環的にじゃんけん状の有利/不利）つきターン制バトルシミュレーター
- 15%確率で発生するランダムハプニング（離脱・強化・会心）
- ポケモン風バトル画面：手続き生成のドット絵モンスター（左右対称・属性ごとの専用パーツ）、HPウィンドウ、攻撃/被弾/瀕死アニメーション
- Canvas描画によるOGP風リザルトカード（PNGダウンロード・Xシェア導線）
- QRバトルカード：キャラをJSON化してQRコード発行、カメラで読み取ると同じキャラを対戦相手として召喚
- バーコード召喚：BarcodeDetector API（対応ブラウザのみ）または手入力で、実在のバーコード数字をハッシュ化して謎のモンスターを生成
- 自己満レベルアップ：`localStorage`にバトル成績を保存、XP・称号解除・連勝記録を追跡
- Xアイコン取得（実験的・非公式）：`unavatar.io`経由でアイコンをベストエフォート表示

## 既知の制約・未検証事項

- カメラを使う機能（QR/バーコード読み取り）は実ブラウザでの動作確認が未実施。Node上ではロジック（QRペイロードの往復整合性、バーコード→モンスター変換の決定性、XP/称号ロジック）のみユニットテスト済み
- iOS Safariは`BarcodeDetector`未対応のため、QRはjsQRで読めるがバーコードは手入力のみになる
- unavatar.io経由のXアイコン取得は非公式な抜け道。安定性・利用規約はグレーゾーンなので本番採用は要検討
- `localStorage`ベースの進行状況は端末・ブラウザ依存（同期なし）

## 元の提案書からの次のステップ案

1. Next.js (App Router) 化し、`@vercel/og` で真のOGP画像を動的生成
2. Supabase/Firebaseでバトル履歴を永続化し、`localStorage`の自己満レベルをアカウント紐付けに移行
3. ステータス/技生成をLLM（gpt-4o-mini や Claude Haiku）経由に置き換え、より多様な生成にする（現在はハッシュ+テンプレートのみ）
4. X（Twitter）連携は公式API課金を避けるなら、現状のQR/バーコード方式かユーザー画像アップロード方式のどちらかに一本化するのが無難
5. バーコードスキャンをQuaggaJS等に載せ替えれば、BarcodeDetector非対応ブラウザでもカメラ読み取り対応可能

## Claude Codeへの引き継ぎ手順

1. `tsubuyaki-battler.html` と本ファイルを同じフォルダに置き、そこで `git init`
2. Claude Codeを起動し、「このHANDOFF.mdを読んで、tsubuyaki-battler.htmlの現状を把握してから次のステップ1を実装して」のように指示する
3. 以降はClaude Code側で通常の開発（テスト・Next.js移行等）を進める
